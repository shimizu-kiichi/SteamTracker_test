>

このドキュメントはフォーム実装者・テスター向けの仕様書です。フォームの入力項目、検証ルール、スプレッドシートへの記録仕様、通知トリガー、受け入れ基準（Acceptance Criteria）および簡易テストケースを定義します。

---

## 1. 前提

- Google Formなどを参考に自作フォームで実装され、送信結果は Google スプレッドシートに自動で追記されます。
- 画像は Google Drive に保存し、ファイル ID または共有リンクをスプレッドシートに保管します。

---

## 2. フォーム項目（実装上の詳細）

各項目について、フォーム上のラベル、内部キー（field name）、型、必須性、検証ルール、UI ヒントを定義します。

| ラベル（フォーム表示） | 内部キー | 型 | 必須 | 検証 / 制約 | UI ヒント |
| --- | --- | ---: | :--: | --- | --- |
| 氏名 | name | 短文 | 必須 | 空欄不可 | 登録者のフルネームを入力してください |
| メールアドレス | email | メールアドレス | 必須 | メール形式のバリデーション | 連絡可能なメールアドレスを入力してください |
| 団体名 | organization | 短文 | 任意 | 文字数上限 100 | 任意（所属があれば入力） |
| 対象の写真ファイル | photo_upload | ファイル | 必須 | MIME: image/jpeg, image/png; 最大 5MB / 1 ファイル (複数可は運用次第) | 物の周囲が分かる写真をアップロードしてください |
| 明け渡し日 | handover_on | 日付 | 必須 | 当日または未来日（過去日は不可） | 例: 2025-12-31 |

### UI 実装メモ

- ファイルアップロードは Drive への保存を行い、そのファイル ID を記録する。公開リンクを作成するか、権限管理で管理者が閲覧できるようにする。
- 日付の入力はカレンダーピッカーを用いる。

---

## 3. スプレッドシートへの記録仕様

スプレッドシートの列と保存内容を次のように定義します。

| 列 | カラム名 | 型 | 説明 |
|:---:|:---|:---|:---|
| A | registered_at | DateTime | フォーム送信時刻（自動記録） |
| B | email | Text | 登録者メール |
| C | name | Text | 登録者氏名 |
| D | organization | Text | 団体名 |
| E | photo_file_id | Text | Drive ファイル ID |
| F | handover_on | Date | 明け渡し日（YYYY-MM-DD） |
| G | days_until_handover | Number | 明け渡し日までの日数（計算列） |
| H | status | Text | active / archived / pending |
| I | admin_note | Text | 管理者備考 |

### days_until_handover の計算例

G2 に以下を入れて下方向にコピーします。

```excel
=INT(F2 - TODAY())
```

## 4. 自動処理（トリガー）

1. onFormSubmit（フォーム送信イベント）
   - スプレッドシートへの追記
   - 申請者への確認メールを送信（テンプレは下記）
2. Time-driven（毎日 1 回）
   - 明け渡し日までの残日数をチェックし、閾値に応じてリマインドを作成／送信（例: 残り 7 日、3 日、当日）
   - 管理者向けアラート（残日数が 0 以下のものなど）を送信

## 5. 通知テンプレート（実装例）

### 申請確認メール（申請者向け）

件名: 【STEAM_tracker】写真の明け渡し申請を受け付けました

本文例:

```
{name} 様

写真の明け渡し申請を受け付けました。
明け渡し予定日: {handover_on}

申請内容を確認し、必要に応じて管理者よりご連絡します。

--
STEAM_tracker
```

### リマインド（申請者向け）

件名: 【STEAM_tracker】明け渡し期限が近づいています（残り {days} 日）

## 6. 受け入れ基準（Acceptance Criteria）

1. 必須項目（name, photo_upload, handover_on）が未入力で送信できないこと。
2. メール形式のバリデーションが行われること。
3. アップロードされた画像は Drive に保存され、E 列にファイル IDが記録されること。
4. 明け渡し日が過去日の場合、フォームはエラーを返すこと。
5. フォーム送信後、申請者に確認メールが自動送信されること。
---

## 7. テストケース（簡易）

- TC-01: 正常フロー — 全項目正常入力 → 送信成功、スプレッドシートに記録、確認メール送信。
- TC-02: メール不正 — invalid.email → バリデーションエラー。
- TC-03: 画像超過 — 6MB の画像 → アップロード拒否。
- TC-04: 過去日指定 — 明け渡し日を過去日に設定 → エラー。
- TC-05: トリガーチェック — 残日数 7 のレコードにリマインドが作成される。

---

## 8. 運用上の注意

- 画像の保存期間・アクセス権・削除ポリシー（例: 申請処理後 90 日で削除）を運用で明確にすること。
- スプレッドシートのタイムゾーン設定は東京 (GMT+09:00)
