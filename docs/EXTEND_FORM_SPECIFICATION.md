# 延長申請フォーム仕様書

## 1. 概要
本仕様書では、スチームコモンズ物品管理システムにおける延長申請フォームの動作仕様を定義する。延長申請フォームは、利用者が撤去予定日の延長を申請するためのWebフォームであり、Apps ScriptとHTMLを用いて実装される。

## 2. 主な機能
1. 延長申請フォームの提供：利用者がアクセスできるWebフォームを提供する。
2. 入力項目の設定：利用者が必要な情報を入力できるようにする。
3. 日付入力：撤去予定日の延長申請をカレンダーのウィジェットで行えるようにする。
4. 申請内容の保存：入力された申請内容をスプレッドシートに記録する。
5. 申請完了通知：申請完了後、利用者に確認メッセージを表示する。

## 3. 入力項目
ログイン画面では、利用者には以下の情報を入力させる。
- メールアドレス：利用者の登録済みメールアドレスを入力する。このメールアドレスを基に物品を参照する。
延長申請フォームには以下の入力項目を設ける。
- 物品リスト：リストには団体名、物品名、延長希望日を表示し、延長したい物品の延長希望日を変更する。

## 4. 日付入力の制限
- 延長希望日の入力欄には、カレンダーウィジェットを使用し、利用者が選択できるようにする。
- カレンダーウィジェットには年度末を上限日として設定し、それ以降の日付は選択不可とする。

## 5. 管理者への通知
- 申請が送信されると、管理者に申請を許可するかどうかの選択肢を提示するLINEメッセージを送信する。
- 通知内容には、利用者の氏名、延長する物品、延長希望日を含める。
- 申請を許可すれば、スプレッドシートの該当データの撤去予定日を更新し、利用者にメールで承認通知を送信する。
- 申請を拒否すれば、利用者にメールで拒否通知を送信する。

## 6. 利用者への通知
- 申請完了後、利用者にメールで申請内容の確認メッセージを送信する。

## 7. 申請内容の保存
- 申請内容は、管理者が許可した場合のみ、スプレッドシートの明け渡し日の列に反映される。
