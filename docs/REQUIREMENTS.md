# プロジェクト要件定義書

---

### 1. プロジェクトの目的

管理者が認可した物品以外は処分する。そのサポートとなるサービスを作成することがこのプロジェクトの目的です。

---

### 2. 利用者と役割

| 役割       | 対象   | 主な権限・操作                                                                                    |
| :--------- | :----- | :------------------------------------------------------------------------------------------------ |
| **管理者** | 教職員 | ・追加された登録情報を通知で受け取る<br>・全登録物品を閲覧・操作する<br>・延長申請を許可/拒否する |
| **利用者** | 利用者 | ・物品を登録する<br>・リマインダーを受信する<br>・延長申請を管理者に申請する                      |

---

### 3. 主要なシステムのフロー

#### 【登録フロー】

1.  **Step 1:** 利用者は物品を登録する。
2.  **Step 2:** 利用者は、以下の情報を手入力する。
    - 氏名、団体名
    - **明け渡し日**: 利用者が自ら日付を設定する。明け渡し日は年度末とする。
    - **物品の写真**: **その物品がどこに置かれているか分かるように、周囲の状況も含めて撮影する。**
    - **規約のチェック**: **規約の確認をチェックして登録する。**
3.  **Step 3:** 登録が完了すると、情報がシステムに保存される。ここで登録者と管理者に通知する。

#### 【事前通告・管理フロー】

1.  **Step 1: 個別のリマインダー**
    - 各物品に設定された**「明け渡し日」が近づくと**、利用者と管理者に通知する。
2.  **Step 2: 利用者の対応**
    - 通知を受け取った利用者は、自身の「明け渡し日」までに物品を片付けるか、管理者に延長の申請をする。
3.  **Step 3: 管理者の対応**
    - 管理者は撤去完了処理をする。
    - 撤去完了が報告されるとそのデータはアーカイブ（記録として保存）される。

### 4. システムの制約条件

このシステムを開発・運用する上での前提条件と、対象範囲を以下に定めます。

#### 業務上の制約

- **利用資格**: 物品を登録できるのは、スチームコモンズの利用者に限る。
- **設置期間**: 物品の設置期間は、登録日から当該年度の末日（3 月 31 日）までを原則とする。
- **登録対象外の物品**: 生もの、危険物、法令に違反するもの、高価な私物（貴重品）は登録を禁止する。

#### 対象範囲

- **対象**

  - 利用者が所有する物品の登録・編集・撤去完了処理
  - 撤去期限が近づいた際のリマインド通知
  - 利用者から管理者への延長申請と、管理者の承認・否認
  - 管理者による登録物品の操作

---

### 5. 用語集

本プロジェクトにおいて、以下の用語はここで定義された意味で用いることとします。

| 用語           | 定義                                                                                                             |
| :------------- | :--------------------------------------------------------------------------------------------------------------- |
| **利用者**     | スチームコモンズに物品を登録・設置する者。                                                                       |
| **管理者**     | スチームコモンズの職員。                                                                                         |
| **明け渡し日** | 利用者が物品を撤去しなければならない最終期限日。原則として年度末に設定される。                                   |
| **撤去完了**   | 管理者と利用者が撤去完了を確認し、管理者が撤去完了確認操作をする。                                               |
| **アーカイブ** | 撤去完了、または管理者によって処分された物品の登録データを、有効な一覧から過去の記録として保管・移動させること。 |
