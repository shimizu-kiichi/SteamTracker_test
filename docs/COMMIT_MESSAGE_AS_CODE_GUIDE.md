# 実践ガイド：計画ファイルも一緒に管理するGit開発フロー
## Step 1: 計画 - 作業の設計図を作成する 📝

まず、取り組むタスクのブランチを作成し、その中で計画ファイルを準備します。ここでは「スプレッドシートから最終行を取得する関数を追加する」というタスクを例に進めます。
1.  **ブランチを作成・移動します。**
2.  **計画ファイルを格納するディレクトリ (.commits) の中にファイルを作成します。**
    ファイルの名前は`半角小文字苗字-YYYYMMDD-hhmm.md`でお願いします。

    ```bash
    mitani-20251016-2042.md
    ```

3.  **計画ファイルにこれから行う実装の計画を詳細に記述します。**
    **`.commits/mitani-20251016-2042.md` の中身:**

    ```md
    add: スプレッドシートから最終行を取得する関数を追加

    # スプレッドシートから最終行を取得する関数

    指定されたシートのデータ範囲から最終行のデータを取得する
    ユーティリティ関数を実装する。

    - `SpreadsheetApp` を使用して対象のシートにアクセス
    - `getDataRange()` と `getLastRow()` を使って最終行を特定
    - 最終行のデータを配列として返す `getLatestRow(sheetName)` 関数を `Utils.gs` に作成
    ```

    ファイルを保存して閉じます。
-----

## Step 2: 実装 - 計画書に従ってコードを書く 👨‍💻

作成した計画ファイル (`.commits/mitani-20251016-2042.md`) を常に参照しながら、**そのファイルに記載されたスコープだけを忠実に実装します。**

-----

## Step 3: コミット - コードと計画書を一緒に記録する 🚀

実装が完了したら、コードと計画書の両方をステージングし、計画書の内容を使ってコミットします。ここがこのフローの核心です。

1.  **実装したコードと、作成した計画ファイルを両方ステージングします。**

2.  **`-F` オプションを使い、計画ファイルの内容をコミットメッセージとして読み込みます。**

    ```bash
    git commit -F .commits/mitani-20251016-2042.md
    ```
    この操作により、以下の2つが同時に行われます。

      * 計画ファイルの内容が、このコミットの**メッセージ**になります。
      * 計画ファイル自体が、このコミットの**変更内容の一部**として記録されます。
-----
## Step 4: プッシュ - リモートに同期する ✅

最後に、作成したコミットをリモートリポジトリにプッシュします。
