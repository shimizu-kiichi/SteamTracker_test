
## コミットメッセージの仕様

```
<type>(<scope>): <subject>
```

### `<type>`: 変更の種類 (必須)

|                |                                  |
| -------------- | -------------------------------- |
| **タイプ**        | **説明**                           |
| **`add`**     | 新機能の追加                           |
| **`fix`**      | バグ修正                             |
| **`docs`**     | ドキュメントのみの変更                      |
| **`style`**    | コードの動作に影響しないスタイルの変更（フォーマット、空白など） |
| **`refactor`** | バグ修正や機能追加ではないコードの改善（リファクタリング）    |
| **`perf`**     | パフォーマンスを向上させる変更                  |
| **`test`**     | テストコードの追加・修正                     |
| **`chore`**    | ビルドプロセスや補助ツールなど、ソースコード以外の雑多な変更   |

### `<scope>`: 変更の範囲 (任意)

変更が影響する範囲を `()` で示します。（例: `auth`, `api`, `ui`）
わからなかったら大丈夫です。

### `<subject>`: 変更内容の要約 (必須)

- 50文字以内の簡潔な説明を **命令形** で記述します。（例: `○○○○を追加`, `□□□□を修正`）
- 文末に句点（。）は付けません。
### 具体例

**良い例 👍**

```
add(auth): ユーザー登録機能を追加
```

```
fix(api): 無効なIDが渡された際のnullエラーを修正
```

**悪い例 👎**

```
fix
```

> → typeがなく、内容が不明
